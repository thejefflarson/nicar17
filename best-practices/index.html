Programming Best Practices

Google all the time.
When dealing with data, strive to make it so one command kicks off all analysis and cleaning.
That command should delete and recreate any database you have from scratch
Keep the data you've collected -- via FOIA, websites, or other sources -- in its original form, never ever change it.
Go row by row when importing and cleaning, but analyze your data in dataframes or tables.
Keep regular expressions simple. Learn parser generators.
You have a super computer. Process your data in parallel.
Use multiple processes when dealing with cpu bound tasks and multiple threads when dealing with network requests or disk i/o.
Learn to think asynchronously.
Treat data and behavior as separate -- when you create an object strive to keep the data immutable.
Strive to minimize side effects or global state, when called with the same data a function should always return the same thing.
Stay Organized: always clean up after yourself.
Use version control.
Never comment out code. Delete it.
Use an editor that you can change through programming.
If you can, buy another computer and use a new operating system for a while.
Learn the defaults of your computing environment before you customize.
Rely on mixins over inheritance.
Prefer events to function calls for interactives.
Write tests for some things.
Use libraries and frameworks. Read that code. Log into the developers chat rooms.
Don't use libraries or frameworks.
Read other people's code.
Have your own personal projects.
Take one project a year and write it in another language.
Deploy using containers.
Less code is better code.
