Programming Best Practices
When dealing with data, strive to make it so one command kicks off all analysis and cleaning.
That command should delete and recreate any database you have from scratch Keep the data you've collected -- via FOIA, websites, or other sources -- it it's original form, never ever change it
Go row by row when importing and cleaning, but analyze the data in dataframes or tables.
You have a super computer. Process your data in parallel.
Use multiple processes when dealing with disk cpu bound tasks and multiple threads when dealing with network requests.
Learn to think asynchronously.
Treat data and behavior as seperate -- when you create an object strive to keep the data immutable.
Strive to minimize side effects or global state, when called with the same data a function should always return the same thing.
Stay Organized, always be cleaning up after yourself.
Use version control.
Never comment out code. Delete it.
Use an editor that you can change by programming.
Learn the defaults of your computing environment before you customize.
Rely on mixins over inheritance.
Prefer events to function calls for interactives.
Write tests for some things.
Use libraries and frameworks
Don't use libraries or frameworks.
Read other people's code.
Take one project a year and write it in another language.
Deploy using containers.
Less code is better code.
